# Ajo Digital Trust Protocol Behavior

This document defines the behavior and rules of the Ajo Digital Trust savings protocol.

The protocol enforces discipline-based savings using blockchain smart contracts.

All rules are enforced by code.

---

## 1. Savings Plans

Users can create savings vaults under two plans:

### Plan 1
6 Month Plan

### Plan 2
12 Month Plan

Users choose their plan duration when creating a vault.

---

## 2. Target Savings

Each user defines their own savings target.

Example:

Target = 500,000 NGN equivalent

Users can choose any target amount depending on their income.

---

## 3. Deposits

Users can deposit at any time.

Deposits are flexible.

Users can:

- Deposit daily
- Deposit weekly
- Deposit anytime

There is no fixed daily deposit requirement.

The main rule is reaching the target before maturity.

---

## 4. Locked Savings

Funds are locked until maturity.

Users cannot withdraw normally before maturity.

This enforces strong savings discipline.

---

## 5. Emergency Withdrawal

Emergency withdrawal is allowed only if:

User has saved at least 50% of their target.

Example:

Target = 500,000  
Minimum saved required = 250,000

---

## 6. Emergency Token Access

Early withdrawal requires purchasing an emergency access token.

Without the token early withdrawal is impossible.

This prevents emotional withdrawals.

The system is rule-based.

---

## 7. Early Withdrawal Penalty

Emergency withdrawals are charged a penalty.

Penalty Rate:

20%

Example:

Withdrawal = 100,000

Penalty = 20,000

User receives:

80,000

Penalty goes to Treasury.

---

## 8. Treasury

The treasury collects:

- Early withdrawal penalties
- Protocol fees
- Group contribution maturity fees

Treasury is used for:

- Protocol sustainability
- Development
- Future liquidity services

Treasury is visible on-chain.

---

## 9. Multiple Vaults

Users can create multiple savings vaults.

Each vault is tracked separately.

Example:

Vault 1 → 6 Month Plan  
Vault 2 → 12 Month Plan

Balances are independent.

---

## 10. Security Principle

Funds are protected by smart contracts.

No manual intervention.

No emotional decisions.

Code enforces trust.

This is called:

Digital Trust Enforcement

---

## 11. Future Features

Planned future features include:

- Group Contributions (Ajo)
- Stablecoin Savings
- Crypto Lending
- Fiat Off-ramp
- Decentralized Savings Vault
